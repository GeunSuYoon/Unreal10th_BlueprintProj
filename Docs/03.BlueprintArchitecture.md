# BlueprintArchitecture

## 블루프린트 구조

### BP_Unit
```
BP_Unit
├ BP_Normal
│ ├ BP_Player
│ └ BP_Enemy
│   └ BP_EnemyElite
└ BP_EnemyBoss
```

- Enemy와 EnemyBoss를 분리한 이유는 같은 적이지만 다른 로직을 가지기 때문이다.
  - Enemy는 플레이어와 에셋을 공유하고 동일한 장비를 착용한다.
  - EnemyBoss는 거대 몹이나 다른 고유 에셋과 패턴을 사용한다. 

### BP_Item

```

BP_Item
├ BP_UnitParts
├ BP_Weapon
├ BP_WeaponParts
├ BP_Consum
└ BP_Other

```

- 아이템의 ID는 다음 규칙을 따른다.
- ABCCC
  - A는 아이템의 카테고리
    - 1은 유닛 파츠
    - 2는 무기
    - 3은 무기 파츠
    - 4는 소모품
    - 5는 기타 아이템
  - B는 아이템의 세부 카테고리
    - 나중에 결정하면 다시 적을 예정
  - CCC는 아이템의 번호
    - 숫자에 큰 의미는 없고 같은 카테고리에서 아이템을 만든 순서로 지을 예정
    - 000번은 테스트용이다.

### BP_ItemContain

```

BP_ItemContainer
├ BP_Inventory
├ BP_WareHouse
└ BP_Box

BP_Shop

```

- BP_ItemContainer는 BPI_ItemContain을 갖는다.
  - 아이템을 옮기고 분리하고 나누는 등의 액션을 취할 수 있다.
- 둘 다 BPI_ItemInteraction을 갖는다.
  - 아이템을 직접 사용하거나 장착하는 등의 액션을 취할 수 있다.
